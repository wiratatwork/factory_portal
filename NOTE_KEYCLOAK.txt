[OpenID Connect = แอปไม่เก็บรหัสผ่านเอง แต่เก็บที่ Identity Provider (IdP) คือ Keycloak
0. OIDC: รหัสผ่านอยู่ Keycloak (IdP)

1. BFF สร้าง state (CSRF), code_verifier (PKCE)
   → บันทึก pending ใน PostgreSQL
   → ตั้ง cookie app1_bff_pending

2. BFF redirect authorization request ไป Keycloak (request เดียว)
   → ส่ง state + code_challenge (ค่า hash ของ code_verifier)
   → Keycloak: เช็ค KC SSO cookie + เก็บ code_challenge

3. Keycloak redirect กลับ Callback
   → ?code=...&state=... หรือ ?error=...

4. Callback: ตรวจ state ตรง pending (CSRF)
   → Token Exchange: ส่ง code + code_verifier
   → Keycloak ตรวจ hash(code_verifier) == code_challenge

5. ได้ access_token, refresh_token, id_token
   → ValidateAccessTokenAsync (verify JWT)

6. เข้ารหัส tokens → สร้าง sessionId → บันทึก PostgreSQL

7. Set cookie app1_bff_session = sessionId
   → redirect Home
   → ครั้งหลัง: cookie → โหลด token จาก DB → validate/refresh]
